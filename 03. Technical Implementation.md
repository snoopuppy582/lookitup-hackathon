# 3. Technical Implementation

## MVP Goal

Build one working loop:

> select a Trusted Source Pack -> search a claim or subject -> return Evidence Cards

The live demo proves the loop. The pitch mockups show the newsroom workflow around it.

## Core Architecture

```txt
Preset Trusted Source Pack
  -> local sample corpus
  -> search index
  -> claim / subject search
  -> Evidence Card ranking
  -> source links
  -> not-found state
```

## Recommended Stack

### Frontend

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui

### Backend

- Python
- FastAPI
- SQLite
- SQLite FTS5 / BM25 search

## Why This Stack

- Next.js gives a polished demo dashboard fast.
- FastAPI is simple for search and ingestion endpoints.
- SQLite is enough for a hackathon source corpus.
- SQLite FTS5 gives full-text search without a separate search engine.
- Python works well for later URL/RSS extraction.

## P0 Source Pack Model

The final demo starts from preset packs.

Example source packs:

- `Breaking News + Fact-Check Pack`
- `France Official Sources Pack`
- `International Wire Pack`
- `Local Authorities Pack`

Each pack contains a small trusted corpus:

- fact-check articles
- wire or broadcaster samples
- official pages
- local fallback text

## P0 Demo Data

### Kamchatka AI Snow

Claim:

> "Kamchatka snow reached the height of a 10-story apartment building."

Evidence Cards:

- OhmyNews: Korean media reported AI-generated Kamchatka snow visuals as real.
- AFP Fact Check: massive Kamchatka snow footage was AI-generated.
- Local sample: real snow was severe, but the viral apartment-height visuals were not supported.

Expected status:

> Trusted sources say: AI-generated visuals spread online. Claim says: 10-story snow was real.

### Macron Title Mismatch

Claim:

> "Emmanuel Macron is the Vice President of France."

Expected status:

> Trusted sources say: President. Claim says: Vice President.

### Not Found

Claim:

> "Samsung acquired OpenAI in 2025."

Expected status:

> Not found in trusted sources.

## Evidence Card UI

Each card must show:

- source name
- source type
- matched quote
- date or timestamp
- URL
- ranking score

## Result States

### Evidence Found

Show matching Evidence Cards.

### Mismatch

Use this wording:

> Trusted sources say X. Claim says Y.

### Not Found

Use this wording:

> Not found in trusted sources.

Missing evidence tells the journalist the selected source pack currently lacks support for the claim.

## P1 AI Summary Rule

The AI summary is optional.

If implemented, it must follow these rules:

- Use only retrieved Evidence Cards.
- Cite each supporting Evidence Card.
- Say `Not found in trusted sources` when evidence is missing.
- Block outside knowledge.
- Block invented facts.

## Technical Claim

Lookitup's technical value is controlled retrieval: the source pack defines the search boundary, and Evidence Cards make the result visible before publication.
