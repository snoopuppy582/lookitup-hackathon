# 3. Technical Implementation

## Technical Claim

LookItUp's technical value is controlled retrieval.

The source pack defines the search boundary. Evidence Cards make the result visible before publication.

## MVP Goal

Build one working loop:

> select a Trusted Source Pack -> search a claim or subject -> return Evidence Cards

The live demo proves the loop. The pitch mockups show the newsroom workflow around it.

## P0 Rule

P0 must work without:

- external API keys
- generation
- production database
- live network dependency

The demo runs from local trusted-source data.

## Core Architecture

```txt
Preset Trusted Source Pack
  -> local trusted-source corpus
  -> local search
  -> claim / subject query
  -> Evidence Card ranking
  -> source links
  -> not-found state
```

## P0 Source Pack Model

The final demo starts from preset packs.

Example packs:

- `Breaking News + Fact-Check Pack`
- `France Official Sources Pack`
- `International Wire Pack`
- `Local Authorities Pack`

Each pack contains a small trusted corpus:

- fact-check articles
- wire or broadcaster samples
- official pages
- local fallback text

## P0 Demo Data

### Kamchatka AI Snow

Claim:

> "Kamchatka snow reached the height of a 10-story apartment building."

Evidence Cards:

- OhmyNews Fact Check: Korean media reported AI-generated Kamchatka snow visuals as real.
- OhmyNews follow-up: KBS and MBC later deleted related AI snow videos.
- AFP Fact Check: massive Kamchatka snow footage circulating online was AI-generated.

Expected status:

> Trusted sources say: AI-generated visuals spread online. Claim says: 10-story snow was real.

### Not Found

Claim:

> "Samsung acquired OpenAI in 2025."

Expected status:

> Not found in trusted sources.

### Backup Mismatch

Claim:

> "Emmanuel Macron is the Vice President of France."

Expected status:

> Trusted sources say: President. Claim says: Vice President.

## Result States

### Evidence Found

Show matching Evidence Cards.

### Mismatch

Use this wording:

> Trusted sources say X. Claim says Y.

### Not Found

Use this wording:

> Not found in trusted sources.

Missing evidence tells the journalist the selected source pack currently lacks support for the claim.

4. Return Evidence Cards.
5. Show source name, quote, date, and URL.
6. Open source links.
7. Show not-found state.
8. Polish the demo UI.

## AI Summary Rule

AI summary is P1.

If implemented, it must:

- use only retrieved Evidence Cards
- cite each supporting Evidence Card
- say `Not found in trusted sources` when evidence is missing
- block outside knowledge
- block invented facts

## Demo KPI

Primary KPI:

> Time-to-evidence.

Measure the demo as:

1. source pack selected
2. claim searched
3. Evidence Cards visible
4. source link opened
5. unsupported claim shown as `Not found in trusted sources`
