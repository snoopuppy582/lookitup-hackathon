# 2. Pitch Deck & Script

## Pitch Goal

In 5 minutes, prove one claim:

> Lookitup creates a source boundary before publication.

The pitch moves through three beats:

1. A claim reached publication before evidence caught up.
2. Journalists need source evidence under deadline pressure.
3. Lookitup turns the trusted-source boundary into Evidence Cards.

## Slide 5 - Solution

### Title

**Lookitup creates the source boundary.**

### Figure

Use `assets/figures/figure-04-core-loop.png`.

### Content

Core loop:

1. Select sources.
2. Search claim.
3. Read evidence.

### Script

A newsroom, desk editor, fact-checking team, or independent journalist prepares a Trusted Source Pack before the deadline.

During reporting, the journalist searches one doubtful claim inside that pack and reads Evidence Cards with source, quote, date, and link.

## Backup Q&A

| Question | Answer |
| --- | --- |
| How is this different from a search engine? | The search boundary is selected before the deadline. Lookitup searches inside that trusted pack and returns Evidence Cards. |
| Is this automatic fact-checking? | It is pre-publication evidence search. The journalist decides. |
| Why use local data in the demo? | The live pitch should not depend on network access. URL/RSS ingestion comes after the core loop works. |
| Does P0 need an API key? | No. P0 runs from local trusted-source data. |
| Does P0 need a database? | No production database. A local corpus or lightweight local index is enough for the demo. |
| What is the technical core? | Controlled retrieval: source packs define the search boundary, and Evidence Cards make evidence visible. |
